The Dev CLI has been created at `cli/dev.py` with launchers `dev.bat` (Windows) and `dev.sh` (Unix).

---

## 1. Commands

| Command | Description |
| --- | --- |
| `dev init` | Create/update `dev.json` (interactive mode) |
| `dev status` | Display the current active configuration |
| `dev sync code` | Sync all languages defined in the config |
| `dev sync code --lang kotlin` | Sync only Kotlin |
| `dev sync code --dry-run` | Preview commands without executing |
| `dev sync doc` | Sync the entire documentation folder |
| `dev sync doc --file path/to/file.pdf` | Sync a specific file |
| `dev sync doc --entity-provider langextract` | Override the default provider |

---

## 2. Usage from New Projects

You can run the CLI from any project directory by referencing the absolute path to the launcher:

**On Windows:**

```bash
# From any project folder:
C:\ai\graph-rag-tiny\dev.bat init        # Create dev.json in the current directory
C:\ai\graph-rag-tiny\dev.bat sync code   # Sync code based on local config
C:\ai\graph-rag-tiny\dev.bat sync doc    # Sync docs based on local config

```

---

## 3. Configuration (`dev.json`)

The `dev.json` file is automatically generated by the `init` command. Each project maintains its own unique configuration, which stores:

* **Neo4j:** URI, username, and password.
* **Qdrant:** URL and embedding model.
* **Code:** List of languages and their respective root paths.
* **Docs:** Documentation folder, entity provider, and GLiNER model.

---

## 4. Environment Management

The CLI automatically detects the virtual environments (`venv`) for `code-tiny` and `doc-tiny` (locating `.venv/Scripts/python.exe`) to ensure commands are executed within the correct environment.
